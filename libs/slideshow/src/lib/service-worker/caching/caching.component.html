Know This

  • Cache API is available in ServiceWorker and main thread
  • Caching is NOT required for ServiceWorker, and ServiceWorker is NOT required for caching, although people closely associate the two them
  • SW Caching enables offline web apps and installable web apps
  • The Cache stores Request/Response object pairs
  • You can create multiple named caches
  • YOU are responsible for clearing out old caches. Objects never expire
  • With Prefetching, parsed javascript is cached
  • AppCache was TERRIBLE
